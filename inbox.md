  - 场景：最近 AI 热度持续上升，希望系统理解核心概念并建立学习框架
  - 记录：
    - 背景：最近 AI 特别火，信息很多但概念容易混在一起，看到名词时常常“听过但讲不清”
    - 核心目标：先建立统一认知，再进入实践；做到“能解释、能比较、能落地”
    - 学习范围：
      - 基础层：LLM、Prompt、Context、Memory、RAG
      - 执行层：Agent、Tool Calling、Workflow、Automation
      - 组织层：  Skills、Rules、Playbook（可复用流程）
    - 重点概念：Agent、Skills（以及它们在工作流中的关系）
    - 我目前最关心的问题：
      - Agent 和普通聊天机器人（Chatbot）本质差异是什么
      - Skill 和 Prompt 的边界在哪里，什么时候该沉淀成 Skill
      - 什么样的任务适合“直接问模型”，什么样的任务适合“交给 Agent 流程”
      - 在个人知识管理场景里，最小可行工作流（MVP）应该长什么样
    - 概念关系（先形成脑图）：
      - Model 提供理解和生成能力
      - Agent 用目标 + 步骤 + 工具调用把能力转化为结果
      - Skills 将高频任务标准化，提升稳定性和复用性
      - Workflow 把输入、整理、产出、回顾串成闭环
    - 学习完成标准（DoD）：
      - 能用自己的话解释 10 个核心术语（每个不超过 3 句话）
      - 能给出 2 组常见易混概念的对比（如 Agent vs Chatbot）
      - 能跑通 1 个个人最小实践：想法输入 -> 自动整理 -> 文件归档 -> GitHub 同步
