## 10 个核心术语速查

1. **LLM（大语言模型）**：负责理解与生成文本的基础能力层。
2. **Prompt**：给模型的任务说明和输出约束。
3. **Context**：当前模型可见的上下文信息（对话、文档、规则等）。
4. **Memory**：系统保存并复用的历史信息，用于连续任务。
5. **RAG**：检索外部知识后再生成答案，提升准确性与可追溯性。
6. **Agent**：围绕目标进行多步执行、可调用工具并迭代修正的系统。
7. **Tool Calling**：Agent 调用外部能力（读文件、跑命令、请求 API 等）。
8. **Workflow**：将输入、处理、产出、回顾连接成稳定流程。
9. **Skill**：可复用的方法模板，约束任务步骤与质量标准。
10. **Rule**：长期生效的行为规范，用于统一风格与边界。

## 30 秒可复述版（Day 1 口播）

- AI 的核心可以分四层：`Model`、`Agent`、`Tools`、`Workflow`。
- `Model` 负责理解和生成，`Agent` 负责按目标拆步骤并执行。
- `Tools` 让 Agent 能读写文件、跑命令、调接口，不再只会“说”。
- `Skill` 是可复用 SOP，`Prompt` 是单次指令；重复任务优先沉淀 Skill。
- `RAG` 解决“信息从哪来”，`Memory` 解决“系统记住什么”。
- 个人落地最小闭环：想法输入 -> 结构化整理 -> 文档归档 -> GitHub 同步。

## 关键关系（避免混淆）

- `Model` 是能力底座，`Agent` 是执行者，`Tools` 是执行手段。
- `Prompt` 偏一次性指令，`Skill` 偏可复用流程模板。
- `RAG` 解决“信息来源”，`Memory` 解决“持续记忆”。
- `Workflow` 关注端到端闭环，`Skill` 关注单任务标准化。

## 常见对比（必须会讲）

### Agent vs Chatbot

- Chatbot 以问答为主，通常一轮一轮回应。
- Agent 以目标为主，能自主拆步骤并执行动作。
- 判断标准：是否能“产出结果文件/执行操作”，而不只是“给建议”。

### Skill vs Prompt

- Prompt：这次要做什么。
- Skill：这类事长期怎么做。
- 判断标准：如果任务重复出现且质量波动大，就应该沉淀 Skill。

## 7 天学习路径

### Day 1：术语建图

- 任务：写出 10 个术语一句话定义。
- 产出：术语速查版 v1。
- 验收：能口头解释任意 5 个术语，不看资料。

### Day 2：概念关系

- 任务：完成 4 组概念关系与差异说明。
- 产出：关系清单（Model/Agent/Tools/Workflow）。
- 验收：能举 1 个真实例子说明“为什么要 Agent”。

### Day 3：示例拆解

- 任务：找 2-3 个真实 Agent 用例进行逆向拆解。
- 产出：每个用例的“输入-步骤-输出”三段说明。
- 验收：能指出每个用例里 Skill 的作用点。

### Day 4：任务分层

- 任务：把自己常见任务分成“直接问模型”和“适合 Agent”两类。
- 产出：任务分层清单。
- 验收：至少筛出 1 个可立即落地的个人流程。

### Day 5：最小实践

- 任务：搭建个人 MVP 流程（想法输入 -> 整理 -> 归档）。
- 产出：1 条完整记录，包含行动项。
- 验收：能在 10 分钟内完成一条从输入到归档的闭环。

### Day 6：接入 GitHub

- 任务：把实践内容提交并推送，形成版本记录。
- 产出：1 次可追溯提交记录。
- 验收：能解释每次提交的意义（新增/优化/复盘）。

### Day 7：复盘与固化

- 任务：复盘一周并升级模板。
- 产出：v2 学习模板与下周计划。
- 验收：列出 3 条保留做法 + 2 条待优化点。
