# Daily Thinking Hub

用于记录日常想法、tips、问题和复盘，目标是可持续积累并方便回查。

## 目录

- `inbox.md`：随手记录，先收集再整理
- `categories/ideas.md`：想法池，包含可行性评估
- `categories/tips.md`：可复用技巧沉淀
- `reviews/weekly-review.md`：每周复盘
- `examples/first-week-example.md`：完整示例（从记录到复盘）

## 快速开始（5 分钟）

1. 在 `inbox.md` 写下今天的 1~3 条记录
2. 挑 1 条可执行想法，迁移到 `categories/ideas.md`
3. 按评分模型打分，判断优先级
4. 写一个「最小下一步」（可在 30 分钟内开始）
5. 周末把结果更新到 `reviews/weekly-review.md`

## 建议流程

1. 日常记录：全部先写入 `inbox.md`
2. 每周整理：将有价值内容迁移到分类文档
3. 可行性评估：对可执行想法统一打分
4. 行动落地：每周挑 1~3 项推进
5. 周期复盘：更新 `weekly-review.md`

## 可行性评分（1~5）

- 价值（Impact）
- 成本（Effort，越省力分越高）
- 紧迫性（Urgency）
- 风险可控性（Risk Control）
- 复用性（Reusability）

总分：

`score = 0.35*价值 + 0.25*成本 + 0.15*紧迫性 + 0.15*风险可控性 + 0.10*复用性`

阈值建议：

- `>= 4.0`：本周执行
- `3.0 ~ 3.9`：排期推进
- `< 3.0`：先观察

## 维护建议

- 每天记录不超过 5 分钟，先完成再优化
- 每周固定时间整理（例如周五下班前 20 分钟）
- 每周最多推进 1~3 个高分想法，避免分散
